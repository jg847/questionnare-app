# Implementation Phases

This folder turns the approved phase plan from the brief into execution documents for implementation.

## Sprint Map

- Sprint 00: Foundation Lock
- Sprint 01: Platform Skeleton
- Sprint 02: Recommendation Core
- Sprint 03: Public Chat Experience
- Sprint 04: Monetization and Access Control
- Sprint 05: Admin Operations and MVP Hardening
- Sprint 06: Post-MVP Enhancements
- Sprint 07: Affiliate Funnel Expansion

## Order of Execution

1. Complete Sprint 00 before starting implementation.
2. Complete Sprints 01 through 05 in order unless a dependency is explicitly re-planned.
3. Do not start Sprint 06 until Sprints 01 through 05 are working end-to-end in a deployable state.
4. Do not start Sprint 07 until Sprint 06 is complete and the post-MVP enhancements remain stable.

- Product scope and sequencing: brief.md
